# ResearchOS 快速开始

ResearchOS 是给 Codex 使用的科研助理框架。你不需要先阅读代码或运行完整工程；把仓库 pull 到本地后，在 Codex 中用自然语言开始即可。

## 1. 第一次打开

在 Codex 中进入 ResearchOS 仓库目录，然后说：

```text
初始化 ResearchOS 本机环境，帮我建立本地私有配置，但不要写入 GitHub 仓库。
```

Codex 应优先检查：

- `configs/*.example.*` 中的配置样例。
- `.gitignore` 是否已排除真实本机配置和科研材料。
- 是否需要在 `%USERPROFILE%\.researchos\` 或仓库本地 `.researchos\` 下生成私有指针。

## 2. 建立课题工作区

准备一个具体课题目录，然后说：

```text
为这个课题建立 ResearchOS 工作区：<课题路径>
```

课题目录用于保存真实成果，例如：

- `.research/`：项目 manifest、运行状态、材料索引和未决问题。
- `01-reading-cards/`：读书卡。
- `02-literature-matrix/`：综述矩阵、阅读计划、PRISMA 状态和缺口分析。
- `03-manuscript/`：论文草稿、方法审查和结果叙事。
- `.paper/`：论文论断、图表、证据和修订记忆。
- `annotations/`：人工批注收件箱。

这些是真实科研材料，默认不放进 ResearchOS 公开仓库。

## 3. 恢复上下文

后续继续工作时，可以直接说：

```text
继续当前课题，先恢复上下文。
```

Codex 会优先读取本地私有项目指针、项目 `.research/`、读书卡、综述矩阵、论文记忆和最近进展文件。

## 4. 常用自然语言入口

```text
为这篇文献生成读书卡。
整理这些读书卡，形成综述矩阵。
根据这个研究缺口判断是否值得做。
审查这个方法设计能否支撑研究问题。
帮我润色这段论文，但不要改变技术含义。
处理我的批注收件箱，并给出可采纳建议。
检查当前课题走到哪一步，下一步应该做什么。
```

## 5. 需要工具时

普通阅读、写作、判断和审查优先由 Codex 直接完成。只有缺少本地语料、需要维护 Zotero 父文档、抽取 PDF 文本、同步阅读总表或生成机器中间产物时，才使用 `tools/` 下的脚本。

任何 Zotero 写入、文件移动、批量改名、删除或外部 API 写入都必须单独确认。

## 6. 发布和隐私

公开 GitHub 仓库只应包含 ResearchOS 的通用框架。真实 Zotero 数据库、全文缓存、读书卡、项目进展、论文材料、本机路径和密钥不得提交。

发布前按 `PRIVACY.md` 和 `docs/github-release-checklist.md` 检查。
