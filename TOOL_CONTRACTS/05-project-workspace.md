# 点子、课题工作区与项目材料治理契约

## 1. 适用工具

- `extract_project_materials.py`
- `create_project_workspace.py`

## 2. 工具目的

本专题工具用于管理点子、课题工作区、项目材料抽取和项目上下文恢复。具体项目目录治理、两级结构收束、项目文档排序和下一阶段行动清单应在该课题项目路径下单独治理，不在 ResearchOS 通用 `tools/` 中保留绑定具体课题的专用脚本。

## 3. 允许行为

- 创建经过用户确认的 ResearchOS 输出目录。
- 根据用户指定项目工作区生成或读取项目上下文材料。
- 只读扫描项目材料并生成治理计划。
- 在用户批准后按计划整理项目文档结构。
- 输出行动清单、阶段记录和两级结构规则记录。

## 4. 禁止行为

- 不默认把 ResearchOS 根目录当作具体科研项目。
- 不在未确认时移动、删除或重命名真实科研材料。
- 不把迁移日期当作点子或课题建立日期。
- 不把项目材料治理结果写入 Zotero。
- 不把具体课题事实提升为 ResearchOS 全局规则。

## 5. 项目上下文恢复

处理“当前课题”“继续上次”“研究进展”“阅读卡”等任务时，应先读取 `docs/modes/AGENTS.local-research.md`，再结合：

- 活跃项目配置。
- 项目登记表。
- 项目 manifest。
- 项目总览。
- 材料索引。
- 阅读计划或读书卡索引。

## 6. 验收标准

- 输出目录符合 ResearchOS 编号和命名规则。
- 项目材料变更有计划、清单和人工确认。
- 点子、课题、项目三类对象边界清楚；点子不写入 ResearchOS 根目录。
- 生成的人读文档自然、可继续推进。

## 7. 点子捕获入口

点子捕获由 `idea-to-research-potential` skill 在项目工作区或平级 `0.Inbox/02-unassigned-ideas/` 中完成。
