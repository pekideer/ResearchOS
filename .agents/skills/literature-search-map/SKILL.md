## 目标

用于把模糊研究兴趣转化为可执行的文献发现路线，帮助用户系统检索和扩展文献。

## 输入

- 研究主题、研究对象或初步问题。
- 已知关键词、种子文献、目标学科或目标数据库。
- 可选：时间范围、文献类型、排除领域。

## 工作流

1. 拆解研究对象、方法、场景、变量和指标。
2. 生成中文关键词、英文关键词和同义词。
3. 根据学科和文献类型建议数据库。
4. 根据任务阶段选择检索宽度：
   - 点子探索期：先给宽检索、中等检索和窄检索三组检索式，并设置“直接相关、机制相邻、方法相邻、场景相邻、背景材料、暂不纳入”分层纳入标准。
   - 研究缺口论证期：再收紧为可复现的核心检索式和明确排除标准。
5. 构造中英文检索式。
6. 设计种子文献策略和引用追踪策略。
7. 明确排除标准，避免检索范围无限膨胀；但点子探索期的排除标准应只排除明显无助于方向判断的材料。
8. 如果本轮实际查阅了网络数据库、出版商页面、Crossref/OpenAlex/Semantic Scholar/Google Scholar 等外部来源，且发现值得读或待进一步确认的文献，必须生成外部候选阅读清单；未实际查阅时只输出检索路线，不编造候选文献。

## 外部候选阅读清单

当产生外部候选阅读清单时，输出 Markdown 和可选 CSV，建议使用 `templates/external-reading-candidates.md` 与 `templates/external-reading-candidates.csv` 的字段。每条候选至少包含：

- `title`、`authors`、`year_or_date`、`publication_or_source`、`doi`、`url`
- `discovered_from`：数据库、出版商页面、搜索引擎或引用链来源
- `search_query_or_page`、`discovered_at`
- `reason_to_read`：为什么值得读，明确是事实、推断还是待核查
- 候选分层：面向用户显示为“直接相关、机制相邻、方法相邻、场景相邻、背景材料、暂不纳入”；机器字段可保存为 `core`、`adjacent-mechanism`、`adjacent-method`、`adjacent-scenario`、`background` 或 `exclude`
- 阅读优先级：面向用户显示为“高、普通、低、暂缓”；机器字段可保存为 `high`、`normal`、`low` 或 `hold`
- `suggested_collection`：建议 Zotero 项目文献集，例如 `00.09-watchlist-待补读与跟踪`、`00.02-review-core-综述核心文献`、`00.04-method-source-方法与模型来源`；如项目覆盖层未建立，应写“建议新建/待确认”
- `suggested_tags`：建议标签，例如 `rs:read/todo`、`rs:priority/high`、`rs:use/review`、主题标签和来源标签
- `duplicate_check`：是否已在 Zotero 父文档或本地候选池中发现疑似重复；未查则写“未核查”
- `next_action`：导入 Zotero、补全文、精读、追踪引用或暂缓

## 输出

- 中文关键词
- 英文关键词
- 同义词
- 数据库建议
- 检索式
- 种子文献策略
- 引用追踪策略
- 排除标准
- 点子探索期的候选分层标准
- 若实际查阅外部来源并发现候选：外部候选阅读清单 Markdown/CSV

## 质量规则

- 检索式应可直接复制到数据库中试用。
- 同义词要区分广义词、窄义词和相邻概念。
- 排除标准必须具体，不能只写“无关文献”。
- 对早期点子，不得只给窄检索式；至少给一组高召回宽检索式和一组可控噪声的中等检索式。
- 不声称某个数据库一定覆盖全部文献。
- 对不熟悉的专门数据库应标注需要用户确认。
- 外部候选清单中的 文献集和标签 只是导入/整理建议，不代表已写入 Zotero。
- 面向人工阅读的说明和表格列名必须遵守 `POLICIES/OUTPUT_LANGUAGE_POLICY.md`：除可复制英文检索式、数据库字段、文献题名、DOI/URL、数据库/软件/API 名、路径/命令/字段名、skill 名、文献专门术语和领域共同认可缩写外，普通概念用中文表达。

## 安全规则

- 不编造种子文献、DOI、作者或数据库覆盖范围。
- 不把未检索验证的结果写成已发现文献。
- 不把外部候选文献写成已精读证据；未读全文前只能作为题录/摘要级候选。
- 不因生成建议文献集/标签 而写入 Zotero；写入必须另行审批。
- 对不确定数据库、术语或学科边界标注“需要核查”。

## 完成条件

- 输出关键词、同义词、检索式、数据库路线和引用追踪策略。
- 检索式可直接复制到目标数据库试用。
- 排除标准具体，且检索范围边界清楚；若任务是点子探索，已说明哪些边缘文献先保留为候选而不是立即排除。
- 若实际发现外部候选文献，已返回题录、DOI/URL、来源、建议文献集/标签、复核状态和下一步行动。
- 不确定数据库或检索字段已标注“需要核查”。
