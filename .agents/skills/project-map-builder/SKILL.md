## 目标

在用户授权的项目文本范围内，构建项目蓝图、当前位置、历史路线和可能下一步。该 skill 关注项目状态理解和过程回溯，不直接生成论文内容，也不替代具体文献阅读或写作 skill。

## 输入

- 用户指定或当前活跃的项目、点子或 Obsidian vault 目录。
- 用户开放读取权限内的文本文件：入口页、索引、研究进展、报告、读书卡、矩阵、批注、`.research/` manifest、`.researchos/` 指针。
- 过程性记录：`PROJECT_STATE.md`、迁移日志、review log、执行报告、最近生成的汇报材料。
- 可选：Zotero 父文档上下文包、项目文献集计划、PRISMA 状态表。

## 工作流

1. 确认项目边界：如果用户说“当前项目”，先读 `docs/modes/AGENTS.local-research.md`、`.researchos/active_ideas.md`、`.researchos/project_registry.yml`；如果用户指定点子目录，先确认其当前项目工作区，再读取点子入口、判断索引、报告索引和阅读卡索引。
2. 建立项目蓝图：主题与目标、证据层、判断层、输出层和过程层。
3. 回溯路线：起点、中间节点和当前位置。
4. 判断当前位置：材料搜集、精读、矩阵、缺口判断、选题凝练、方法设计、写作、返修或治理阶段。
5. 推测未来路线：基于已有证据和用户意图，给出 1-3 条可能路线，并区分事实与推断。
6. 输出项目地图，不写入文件，除非用户明确要求保存。

## 输出结构

- 项目/点子一句话定位。
- 当前蓝图：目标、证据、判断、输出、过程记录。
- 从起点到当前位置的路线。
- 当前所在阶段。
- 未决问题和缺口。
- 可能下一步路线。
- 需要读取或补足的材料。

## 质量规则

- 只读取用户开放权限内的文本材料。
- 不把现有项目材料没有写成领域没有。
- 不编造项目历史、用户意图、文献状态或执行结果。
- 推测下一步时必须标注基于当前材料的推断。
- 不把机器中间产物当作人读结论；机器产物只能作为索引或证据线索。

## 完成条件

- 已说明读取了哪些项目入口或过程记录。
- 已形成项目蓝图、路线回溯、当前位置和下一步建议。
- 已区分事实、推断、建议和需要核查项。
- 输出能帮助后续 `semantic-route-planner` 或 `report-context-navigator` 接续使用。
