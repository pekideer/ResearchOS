## 目标

用于在论文写作和返修前建立 `.paper/` 机器可读 记忆，减少多轮修改中 论断、图号、证据、数值和审稿回复位置漂移。

## 输入

- 论文草稿、摘要、引言、结果、讨论、结论或返修稿。
- 图表清单、图注、结果表、统计结果、补充材料或用户说明。
- 可选：审稿意见、已有 论断-证据 表、修改记录、目标期刊要求。

## 工作流

1. 识别稿件章节和关键段落，生成 `manuscript_map.yml`。
2. 抽取主要 论断，生成 `claims.yml`，每条 论断 保留位置、证据状态和风险等级。
3. 抽取 图/表，生成 `figures.yml`，记录图号、回答的问题、支持的 论断 和不能推断的内容。
4. 抽取 证据材料，生成 `evidence_artifacts.yml`，记录数据、图表、统计检验、文献引用或理论推导。
5. 如处于返修阶段，生成或更新 `revision_history.yml`。
6. 后续 `claim-evidence-audit`、`academic-polishing`、`results-figure-narrative`、`reviewer-response` 应优先读取 `.paper/` 记忆，再读取全文。
7. 不确定字段用 `?`，不得补造数值、图号、实验结果或引用。

## 输出

- `.paper/manuscript_map.yml`
- `.paper/claims.yml`
- `.paper/figures.yml`
- `.paper/evidence_artifacts.yml`
- `.paper/revision_history.yml`
- 可选：`paper_memory_summary.md`

## 质量规则

- 每个 论断 应有稳定 `claim_id`、稿件位置、论断 文本、证据链接 和风险等级。
- 每个 图/表 应有稳定 `figure_id` 或 `table_id`，并说明支持哪些 论断。
- 每个 证据材料 应说明来源、类型、可复查位置和限制。
- 记忆 是索引，不替代原始稿件或原始数据。
- 不确定、缺失或待核查字段必须用 `?`。

## 安全规则

- 不编造图表、数据、统计显著性、引用或审稿意见。
- 不把没有证据的 论断 标为已支持。
- 不改变论文技术含义。
- 不把 `.paper/` 记忆 当作事实来源；它只是指向原始证据的索引。

## 完成条件

- `.paper/` 记忆 的核心文件结构已给出或可直接保存。
- 论断、图/表 和 证据材料 能相互引用。
- 后续写作类 skill 能根据 记忆 定位稿件位置和证据状态。
- 所有不确定字段已标注 `?` 或“需要核查”。
