# 工程类论文逻辑

## 核心结构

工程类论文通常需要清楚回答：

1. 研究对象是什么？
2. 问题边界是什么？
3. 关键变量和约束是什么？
4. 方法如何作用于对象？
5. 指标如何衡量效果？
6. 验证路径能否支持结论？

## 对象

研究对象需要具体到系统、设备、工况、过程、算法对象或应用场景。避免只写宽泛领域名。

## 边界

边界包括：

- 适用场景
- 数据范围
- 模型假设
- 工况范围
- 控制条件
- 不处理的问题

## 变量

- 自变量：主动改变或比较的因素。
- 因变量：被观测或评价的结果。
- 控制变量：需要保持一致或纳入控制的因素。
- 干扰变量：可能影响结果但无法完全控制的因素。

## 指标

指标必须对应研究问题。常见风险包括：

- 指标只反映局部性能，无法支撑总体结论。
- 指标与用户关心的工程目标不一致。
- 缺少成本、鲁棒性或可实施性指标。

## 验证路径

验证路径可以包括：

- 理论分析
- 仿真
- 实验
- 案例研究
- 对照组比较
- 消融实验
- 鲁棒性和敏感性分析

结论不能超过验证路径支持的范围。
